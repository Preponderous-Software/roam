⛏ Cave Generation System

Design document for Roam's underground cave system — procedural generation, vertical traversal, depth-scaled loot, and multi-level exploration.

Overview

The cave system adds a vertical (z-axis) dimension to Roam's existing 2D tile world. The surface world (z = 0) is unchanged. Caves exist at z = −1, −2, and −3, each progressively darker, more dangerous, and richer in rare ore.

Players discover a Cave Entrance entity in the world (most commonly in mountain and forest rooms). Interacting with it descends into the cave below the same (x, y) map coordinate. A Cave Ladder inside every cave room leads back up. Deeper cave levels contain additional entrances to go further down — up to Abyss depth.

Cave Generation Algorithm

Each cave room uses a cellular automata (CA) smoothing pass to carve natural-looking corridors from a random initial state. The algorithm runs entirely at room-generation time; no pre-planning across rooms is needed.

Steps

  • Fill with Stone. Every tile in the grid is initialised as a solid Stone entity.

  • Random seeding. Each cell is cleared (made open floor) with probability p_open. Default: 45 % at z=−1, 40 % at z=−2, 35 % at z=−3 — deeper levels are more enclosed.

  • Cellular automata smoothing (4 iterations). For every cell, count the number of stone cells among its 8 Moore-neighbourhood neighbours. If the count ≥ 5, the cell becomes (or stays) stone; otherwise it becomes open. The pass runs on a copy of the grid so cells don't interfere with each other.

  • Flood-fill connectivity check. BFS from the centre tile identifies the largest connected open region. All open cells outside the largest region are sealed back to stone, preventing disconnected pockets.

  • Guarantee centre access. If the centre tile ended up as stone (rare but possible), the centre and its four cardinal neighbours are force-cleared to create a player entry point.

  • Place Cave Ladder. A single CaveLadder entity is placed at a random open cell inside the largest connected region — always reachable by the player.

  • Scatter ores. Each open or open-adjacent cell has a small depth-scaled chance to spawn ore (coal → iron → gold as depth increases). Ore sits inside stone cells so the player must have mined/cleared a path to reach it.

  • Optional deeper entrance. If z > −3, a CaveEntrance is placed at a second random open cell far from the ladder, giving access to the next depth level.

Architecture Changes

The cave system extends the existing room-based world with a third dimension (z) while keeping full backward compatibility with existing saves.

Z-axis dimension in Map

Map._roomIndex changes from {(x, y): Room} to {(x, y, z): Room}. All methods (hasRoom, getRoom, generateNewRoom, addRoom, consumeIsNewRoom) gain an optional z=0 parameter. Surface rooms continue to work at z = 0 with no code changes in callers that don't pass z.

Z-axis in Room

Room.__init__ gains a z=0 parameter and a getZ() accessor. This lets WorldScreenPersistence.saveRoomToFile derive the correct file path from the room itself rather than needing a separate z argument.

Save-file path for cave rooms

Config.getRoomFilePath(x, y, z=0) returns room_x_y.json for z = 0 (preserving old saves) and room_x_y_z.json for z ≠ 0. No migration of existing saves required.

Player depth tracking in WorldScreen

A new field self.currentZ = 0 tracks the player's depth. The methods _descend() and _ascend() move the player between levels, saving the current room, loading or generating the target room, and placing the player at the target entry point (nearest open cell to the room centre).

Depth is persisted in playerLocation.json as an optional roomZ field (defaults to 0 for old saves). When underground, a small depth indicator is drawn in the HUD.

Cave room generation in RoomFactory

Map.generateNewRoom delegates to RoomFactory.createCaveRoom(x, y, z) when z < 0. The cave factory runs the CA algorithm and populates ore, ladder, and optional deeper entrance. Surface generation (z = 0) is entirely unchanged.

Player Experience

Discovery
Finding Caves

Cave Entrances appear as solid structures in mountain (40 %), forest (15 %), and grassland rooms (5 %). The codex logs each cave entity discovered, rewarding exploration.

Navigation
Descending & Ascending

Walk adjacent to a Cave Entrance or Ladder and use the place action (right-click / F key). The HUD immediately shows the new depth level.

Exploration
Cave Movement

Cave corridors funnel movement along carved paths. Rooms connect horizontally at each depth level (walk off an edge → adjacent cave room). The existing push-stone mechanic lets players excavate blocked routes.

Loot
Depth-Scaled Ore

Shallow caves hold coal and iron. Deep caves add gold. The Abyss concentrates gold and sets the stage for future gem types. Players mine ore via the gather action as on the surface.

HUD
Depth Indicator

A small "Depth: X" label appears in the corner when underground. Hidden at the surface to keep the HUD uncluttered.

Persistence
Save & Load

Cave rooms are saved to disk on the same room-transition trigger as surface rooms. Player depth is stored in playerLocation.json and restored on load.

Future Extensions

  • Cave creatures — bats, cave spiders, or blind fish as new living entities with unique drops
  • Gem ore — a fourth ore tier (diamond / ruby) exclusive to the Abyss
  • Cave water — pools that slow movement and flood passages over time
  • Mushrooms & cave plants — new food sources underground that glow faintly
  • Torches matter — link day-night overlay to depth so deep caves are pitch-black without placed torches
  • Structural caves — pre-built ruins at rare spawn points with loot chests
  • z-aware minimap — separate minimap layer per depth, toggled with a key
